How It Works

A plain-English guide to the Safe Bike Lanes RVA detection system.

The Big Picture

This project uses a bicycle-mounted camera and GPS tracker to record rides, then automatically scans the footage for common bike lane hazards — things like vehicles blocking the lane, leaves covering the surface, and sediment washed across the path. Every hazard that gets found is pinned to a real GPS coordinate so you can see exactly where problems are on the map.

Step by Step

  1. Record the ride. A camera mounted to the bike records video while a GPS device logs the route as a GPX file — a simple format that stores coordinates and timestamps at regular intervals.
  2. Upload both files. The video and GPX file are uploaded through a form on this site. They get stored in Supabase, a cloud database and storage service, ready to be processed.
  3. Trigger AI analysis. The backend sends the files to Modal, a cloud platform that provides on-demand GPU computing. A computer vision model called YOLOv8 (You Only Look Once, version 8) scans every frame of the video and draws bounding boxes around anything it recognises as a hazard.
  4. Match detections to GPS. Each video frame has a timestamp. The system syncs those timestamps with the GPX file to figure out exactly where on the route the camera was when each hazard appeared. That gives every detection a real-world latitude and longitude.
  5. Save to the database. Each confirmed hazard — its type, location, confidence score, and timestamp — is written to a Supabase database table called detections.
  6. View on the map. The map page reads from that table and places a dot for every detection. You can filter by hazard type or date range to focus on what matters to you.

The AI Model

YOLOv8 is a real-time object detection model. It was originally trained on a general dataset (people, cars, animals, etc.) and then fine-tuned here on labelled images of bike lane hazards. When it looks at a video frame it assigns each detected object a confidence score — how sure it is about the label. Only detections that meet a minimum confidence threshold and appear within the bike lane region of the frame are recorded.

GPU computing on demand. Running a video through a neural network frame-by-frame is computationally intensive — it would take a very long time on a regular laptop. Modal spins up a powerful GPU in the cloud, runs the analysis in minutes, and shuts the machine down again. This keeps costs low and avoids needing dedicated hardware.

Tech Stack

Flask (Python) handles the web backend and API routes, deployed on Render. Supabase stores files and the detections database. Modal runs the YOLOv8 inference on GPU. Leaflet.js renders the interactive map in the browser. The public-facing website is hosted on Hostinger.
