How It Works

A plain-English guide to the Safe Bike Lanes RVA detection system.

The Big Picture

This project uses a bicycle-mounted camera and GPS tracker to record rides, then automatically scans the footage for common bike lane hazards — things like vehicles blocking the lane, leaves covering the surface, and sediment washed across the path. Every hazard that gets found is pinned to a real GPS coordinate so you can see exactly where problems are on the map.

Step by Step

  1. Record the ride. A camera mounted to the bike records video while a GPS device logs the route as a GPX file — a simple format that stores coordinates and timestamps at regular intervals.
  2. Upload both files. The video and GPX file are submitted through a form on this site and stored in the cloud, ready to be processed.
  3. AI analysis runs in the background. Uploading triggers a background job on a cloud GPU. A computer vision model scans every frame of the video and traces a precise outline around anything it recognises as a hazard. Each outline is then checked to confirm it actually overlaps the bike lane area of the frame — not just something nearby.
  4. Match detections to GPS. Each video frame has a timestamp. The system syncs those timestamps with the GPX file to figure out exactly where on the route the camera was when each hazard appeared — giving every detection a real-world latitude and longitude.
  5. Save to the database. Each confirmed hazard — its type, location, and timestamp — is written to a database as a new record.
  6. View on the map. The map reads from that database and places a marker for every detection. Filter by hazard type or date range to focus on what matters to you.

How Detection Works

The AI model used here is a segmentation model — meaning it traces a precise polygon outline around each detected object rather than just drawing a rectangle around it.

That precision matters because the system uses the outline to check whether a hazard genuinely overlaps with the bike lane region of the frame. A vehicle parked just outside the lane won't trigger a detection — only objects whose outline actually intersects the lane area are recorded.

The model was trained on labelled images of bike lane hazards and assigns each detection a confidence score. Only detections above a minimum threshold that also overlap the bike lane get saved.

Why a cloud GPU? Scanning a full video frame-by-frame is computationally intensive — it would take a very long time on a regular laptop. The system uses on-demand GPU computing in the cloud: a powerful machine spins up, runs the analysis in minutes, then shuts back down. No dedicated hardware needed.

Tech Stack

Flask (Python) handles the web backend, deployed on Render. Supabase handles cloud file storage and the detections database. GPU inference runs on Modal. The interactive map is rendered with Leaflet.js. The public website is hosted on Hostinger.
